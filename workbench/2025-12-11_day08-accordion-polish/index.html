FAQ Accordion Component

Production-ready, accessible accordion built with semantic HTML, modern CSS design tokens, and optimized JavaScript. Portfolio-grade component following WCAG 2.1 AA standards.

This is a production-ready FAQ accordion component built as part of a portfolio project. It demonstrates semantic HTML5, modern CSS architecture with design tokens, vanilla JavaScript best practices, and full accessibility compliance (WCAG 2.1 AA).

The accordion follows a "one panel always open" pattern—when you click a closed panel, it opens and closes all others. This ensures users always have access to some content. The first panel opens automatically on page load for better UX.

Full ARIA implementation with aria-expanded, aria-controls, aria-labelledby, and role="region". Closed panels use the hidden attribute to remove them from the accessibility tree. Keyboard navigation is fully supported (Tab, Space, Enter), and focus states are clearly visible with :focus-visible.

Design tokens (CSS custom properties) create a consistent design system across the entire project. They make maintenance easier—change one variable and it updates everywhere. They also enable theme switching (light/dark mode) with minimal code. This component uses tokens for colors, spacing, typography, and animation timing.

Built as part of the html-css-js-portfolio project.
